Features List

This is an accumulated list of the features we put thought into designing and implementing

User Management

User Signup Flow

  • Two well-placed registration forms make it easier and more likely for users to sign up
    • A registration form on the bottom of the front page
    • A registration form accessible from a "REGISTER" button in the navbar
  • Additionally, Sign In form has a link nearby to Sign Up and vice versa

Account Lockout

  • Users are locked out after 3 unsuccessful attempts
  • Super Admins see a special version of the USERS page which allows them to unlock accounts

User Roles

  • Super Admin - Can perform all actions on the site regardless of sale
  • Admin - Can perform all actions on that sale
  • Seller - Can update the catalog, view financial records, create transactions, and view and print tags of that sale
  • Bookkeeper - Can view financial records of that sale
  • Cashier - Can create transactions of that sale
  • Clerk - Can view and print tags of that sale
  • Guest - Can view all open sales

Managing Users in a Sale

  • Admins can assign users a role on a sale
  • Users are by default guests unless they are added by an Admin
  • Admins can revoke roles assigned to users

Inventory Management

Tag printing

  • Print a tag for an individual item
  • Print all tags for a sale
  • CSS creates proper page breaks when printing tags

Transaction/Financial Management

QR Code Checkout System

  • A mobile device can be used to scan items in a transaction
  • Fully tested with all of the top iOS QR apps, leading to the recommendation of Scanbot
  • This works without requiring typing any passwords on the device
  • The mobile device authenticates itself by scanning a QR code on the transaction page

Financial Report

  • A copy of relevant item data is stored with the transaction so that modifying or deleting an item does not affect the financial report

Security

  • Cookies are implemented using Java Play's session feature
  • Authorization checks are done on page requests to verify the user has permission to perform an action or view a page
  • Passwords are hashed using Whirlpool hashing algorithm

Overall Design

  • Utilized Bootstrap for buttons, navbar, tables, and general layout
  • Developed custom CSS code
  • Implemented asynchronous loading of page content with Javascript in several instances, such as search queries
  • Custom 404 Not Found error page
  • Custom message when sales list is empty or query returns no results

Other Features

  • Website utilizes MySQL database
  • Website and Database are bundled into Centos 7 Linux virtual machine
  • Help page guides new users through using the website
}
